# Software Engineer: Giesecke+Devrient (G+D)

During my tenure at G+D, I worked across diverse domains, including biometric hardware integration, telecommunications marketing platforms, and enterprise-grade test automation frameworks.

## 🏗️ Key Projects & Engineering Solutions

### 1. Biometric Android Application (MWC Showcase)
I was the lead developer for a custom Android application integrated with proprietary fingerprint-sensing hardware. The project was a high-profile demonstration for the Mobile World Congress (MWC).
* **Hardware-Software Bridge**: Collaborated with a hardware subsidiary to integrate a Java-based serial communication library with a custom Android UI.
* **Full-Stack Android Implementation**: Designed and implemented the entire user management system, CRUD operations, and media playback mocked/non-mocked data services (resembling an "Android Auto" experience).
* **UX Collaboration**: Worked closely with UX designers to refine the interface for high-stakes public demonstrations.

### 2. Telecom Marketing & Campaign Management
I contributed to a large-scale Java microservices platform used by major telecommunications providers to manage mobile messaging marketing campaigns.
* **Microservices & Web Development**: Developed and maintained CRUD-based microservices using **Java** and built interactive frontend components using **Angular**.
* **Quality Assurance**: Implemented frontend testing suites (Jester) to ensure the reliability of campaign management dashboards used by enterprise-level clients.

### 3. Cucumber-Selenium Test Automation Framework
My most significant architectural contribution was the development of a custom testing framework designed to bridge the gap between high-level QA requirements and low-level browser automation.
* **Selenium-Cucumber Integration**: Engineered a framework that mapped Selenium-driven browser steps to Cucumber step definitions, allowing non-technical stakeholders to build test cases via a web-based reporting and QA tool.
* **Step Parameterization Solution**: I identified a major usability bottleneck in the initial design—the proliferation of static, single-purpose steps. I proposed and implemented a parameterization engine, allowing steps to accept dynamic inputs (e.g., "Sign up as {user}" instead of "Sign up as TestUser1").
* **Scalable Test Logic**: By allowing variables to be injected into complex flows like login and registration, I significantly reduced the framework's maintenance overhead. The framework was successfully adopted across multiple projects.
